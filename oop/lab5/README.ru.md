# Лабораторная 5 — Плагины (функциональность), вариант 3 (Шифрование)

Развивает лабораторную 4. Хост-приложение получило универсальный
конвейер преобразования данных, к которому подключаются плагины — и
перед сохранением в файл, и после загрузки. Вариант, выпавший на
номер 12: `(12 mod 5) + 1 = 3` → **шифрование/дешифрование**.

## Две системы плагинов рядом

| Система       | Что добавляет           | Где живут            | Формат |
|---------------|-------------------------|----------------------|--------|
| Иерархия      | новые классы транспорта | `plugins/*.json`     | JSON   |
| Функциональная| шаги шифрования         | `funcplugins/*.json` | JSON   |

Обе папки сканируются автоматически при запуске; их можно
перезагрузить или просмотреть из GUI. Чтобы добавить новый плагин —
достаточно одного файла, без пересборки и правок кода.

## Встроенные алгоритмы

С хостом поставляются три обратимых алгоритма. Каждый — это
маленькая стратегия, которая регистрируется через `init()`:

| ID        | Название            | Параметры     |
|-----------|---------------------|---------------|
| `xor`     | XOR Cipher          | `key`         |
| `caesar`  | Caesar (byte) Cipher| `shift`       |
| `aes-cfb` | AES-256-CFB         | `passphrase`  |

Три демо-дескриптора создают экземпляры этих алгоритмов с конкретными
значениями параметров:

* `funcplugins/xor-default.json`
* `funcplugins/caesar-7.json`
* `funcplugins/aes-strong.json`

Можно добавлять любое количество дополнительных дескрипторов: например,
плагин с Цезарем +13 и плагин с AES, использующий другой пароль. Меню
«Plugin settings…» включает/выключает каждый плагин и редактирует его
параметры в рантайме — это и есть бонусная часть задания на 10 баллов.

## Конвейер

```
Сохранение:
  объекты ---Marshal---> текстовые байты
                              |
                              v
                       плагин #1.Encode
                              |
                              v
                       плагин #2.Encode  (в порядке регистрации)
                              |
                              v
                       сырые байты ---> файл

Загрузка:
  файл ---> сырые байты
                              |
                              v
                       плагин #2.Decode  (в обратном порядке)
                              |
                              v
                       плагин #1.Decode
                              |
                              v
                       текстовые байты ---Unmarshal---> объекты
```

## Сборка и запуск

```
cd lab5
go mod tidy
go run ./cmd/app
go run ./cmd/app -funcplugins=other/dir
```

Готовая сборка: `bin/lab5/lab5.exe` (рядом лежат папки `plugins/` и
`funcplugins/`).
